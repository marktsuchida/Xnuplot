class xnuplot.Plot([autorefresh=True, persist=False, description=None, kwargs...])¶

An interface to a 2D Gnuplot plot that remembers (and allows modification of) the current plot items.

Parameters:
  • autorefresh (bool) – whether to automatically replot when plot items are added, removed, or replaced or the plot is otherwise modified
  • persist (bool) – whether the plot window (if any) should stay open after Gnuplot quits (i.e. after the Plot object is deleted)
  • description (str) – an arbitrary description that is saved with the plot
  • kwargs – additional keyword arguments to the lower-level Gnuplot or RawGnuplot constructors.

Plot inherits from list. You can use the standard sequence methods (insert(), append(), pop(), and others), the del statement, and the indexing operator ([]) to add, remove, and replace plot items (analogous to the comma-separated arguments to Gnuplot’s plot command).

If the autorefresh attribute is set to True, simply changing the contents of the Plot container will automatically update the plot.

Plot items can be

  • a string (e.g. "sin(x)"),
  • an PlotData instance, or
  • a tuple, (data [, options [, mode]]), which is used as the arguments to construct a PlotData object, where
    • data can either be a string in a format that Gnuplot understands, or an object that exposes binary data (also must be in a format handled by Gnuplot) through the buffer protocol,
    • options is a string containing Gnuplot plot options (e.g. using and with clauses), and
    • mode is one of "file" or "pipe" (default is "pipe"; see Gnuplot.plot() for details).

For example,

plot = xnuplot.Plot()
plot.append("sin(x) notitle")
plot.append(("0 0\n3.14 1.0",
             "using linespoints title 'Line segment'"))

This is roughly equivalent to the following Gnuplot command:

plot sin(x) notitle, \
     'line.dat' using linespoints title 'Line segment'

where the contents of line.dat is

0 0
3.14 1.0

Except that with Xnuplot, you can alter the plot items without having to reenter the unchanged ones:

plot.pop(0) # This removes the sin(x) but keeps the line segment.

Note

Plot objects always use Gnuplot’s plot command to render the plot. To plot bivariate data (using the splot command), use the analogous SPlot class.

autorefresh¶

If True, refresh the plot (reissue the Gnuplot plot command) every time the plot is modified.

description¶

A description for the plot. This can be set to any value, and is saved together with the plot by the save() method.

size¶

Used when part of a Multiplot to set the size of this plot. A pair of scale factors (x, y).

origin¶

Used when part of a Multiplot to set the origin of this plot. A pair of screen coordinates (x, y).

__call__(command[, dataitems...])¶

Execute a Gnuplot command.

Parameters:
  • command (str) – the command to execute
  • dataitems (keyword arguments) – data to pass to Gnuplot (see RawGnuplot.__call__())
Returns:

the output from the command

For example,

plot = xnuplot.Plot()
plot("set style data linespoints")

Note

It is usually not necessary to use the low level data passing interface provided by the dataitems arguments. Instead, use the sequence methods (append(), etc.) to plot data and fit() to perform curve fitting.

interact()¶

Interact with the Gnuplot command line.

This is handy during interactive use for quickly setting or inspecting various plot attributes, reading the Gnuplot online documentation, or for using the mouse to manipulate the plot (when using a mouse-enabled terminal such as the x11 terminal).

Type Control-] to exit interact mode and return to Python.

refresh()¶

Draw or redraw the plot with the current plot items.

save(file)¶

Save the plot to a file.

Parameters:file (string (filename) or file object) – the file to save to.

The saved file will contain the current settings of the attached Gnuplot process as well as the plot items contained in the Plot object. Saved plots can be loaded with load() or by using the xnuplot command line tool.

clone()¶

Create a duplicate Plot object, sharing the same plot items but with its own Gnuplot process. Gnuplot settings and variables are preserved.

close()¶

Terminate the attached Gnuplot process. This is called automatically when the Plot object is deleted (or garbage-collected).

fit(data, expr, via[, ranges, limit, maxiter, start_lambda, lambda_factor])¶

Perform curve fitting using Gnuplot’s fit command.

Parameters:
  • data (tuple or PlotData) – a plot item specifying the data to fit to
  • expr (str) – the Gnuplot expression for the function to fit
  • via (str, tuple, or dict) – the variables to float (and, optionally, their initial values): e.g. "a, b", ("a", "b"), or {"a": 0.1, "b": 3.0}
  • ranges (str) – the range(s) to fit over (this is passed verbatim to Gnuplot): e.g. "[0.0:1.0]"
  • limit – set Gnuplot’s FIT_LIMIT
  • maxiter – set Gnuplot’s FIT_MAXITER
  • start_lambda – set Gnuplot’s FIT_START_LAMBDA
  • lambda_factor – set Gnuplot’s FIT_LAMBDA_FACTOR
Returns:

(params, errors, log), where

  • params is a dictionary containing the fit parameter values,
  • errors is a dictionary containing the errors of the fit for each parameter, and
  • log is the human-readable report from the fit command.
class xnuplot.SPlot([autorefresh=True, persist=False, description=None, kwargs...])¶

An interface to a Gnuplot bivariate (surface) plot.

Usage is exactly the same as Plot, except that the items are plotted using Gnuplot’s splot command instead of the plot command.

class xnuplot.Multiplot([autorefresh=True, persist=False, description=None, kwargs...])¶

An encapsulation of Gnuplot’s multiplot facility (see set multiplot in the Gnuplot manual).

The constructor parameters are the same as with the Plot and SPlot classes, with whom Multiplot shares the following attributes and methods: autorefresh, description, __call__(), interact(), refresh(), save(), clone() (see below), and close().

Multiplot also inherits from list, but as items holds whole Plot and SPlot instances (as subplots), rather than plot items. When the subplots are modified, the change is reflected in the multiplot (if autorefresh is set to True on the multiplot).

The position and scaling of the subplots are determined by the subplots’ origin and size attributes.

Multiplot.save() will save the multiplot together with all of its subplots.

clone([recursive=False, kwargs...])¶

Clone the multiplot. If recursive is True, each subplot is cloned and placed in the new multiplot; otherwise, the cloned multiplot shares all of its subplots with the original.

class xnuplot.GridMultiplot(rows, cols[, rowsfirst=True, upwards=False, title=None, scale=None, offset=None, kwargs...])¶

An encapsulation of Gnuplot’s multiplot facility using a grid layout.

This is a version of Multiplot that uses Gnuplot’s set multiplot layout option.

Parameters:
  • rows (int) – number of rows in the multiplot grid
  • cols (int) – number of columns in the multiplot grid
  • rowsfirst (bool) – if true, fill the grid with subplots in row-first order
  • upwards (bool) – if true, start at the bottom row (or the bottom cell of the first column); else, start at the top row
  • title (str) – add a title for the multiplot
  • scale (tuple) – scale factors (xscale, yscale) applied to each subplot
  • offset (tuple) – offset (in screen coordinates) applied to each subplot

GridMultiplot shares all methods with Multiplot, but has additional attributes corresponding to the constructor arguments.

rows¶

The number of rows in the multiplot grid.

cols¶

The number of columns in the multiplot grid.

rowsfirst¶

If true, the grid will be filled with subplots in row-first order; else, in column-first order.

upwards¶

If true, the grid will be filled from bottom to top, starting with the bottom row (or the bottom cell of the first column, if rowsfirst is True); else, the grid will be filled from top to bottom.

title¶

The title for the multiplot.

scale¶

A pair of scale factors (x, y) that are applied to each subplot.

offset¶

A pair of offsets (x, y) that are applied to each subplot.

xnuplot.load(file[, autorefresh=True, persist=False, class_=None])¶

Load a plot archived by the Plot.save() or SPlot.save() method.

Parameters:
  • file (string (filename) or file object) – the file containing the saved plot to load
  • autorefresh (bool) – whether to automatically replot when plot items are added, removed, or replaced or the plot is otherwise modified
  • persist (bool) – whether the plot window (if any) should stay open after Gnuplot quits (i.e. after the Plot object is deleted)
  • class_ (class) – the class to use for the returned plot object (defaults to Plot, SPlot, Multiplot, or GridMultiplot, depending on the contents of file)

The class_ argument is useful if, for example, you want to load a saved plot as a numplot.Plot object.

xnuplot.closeall()¶

Close all xnuplot plots and terminate all interfaced Gnuplot processes.

exception xnuplot.FileFormatError¶

Raised by load() when the given file does not have the expected format.
